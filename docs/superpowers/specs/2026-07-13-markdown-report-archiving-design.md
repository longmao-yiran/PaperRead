# Markdown 论文阅读报告归档设计

## 目标

更新项目根目录的 `AGENTS.md`，使论文阅读任务仅在用户主动要求时，保存一份与飞书文档内容一致的 Markdown 阅读报告。

## 规则位置

采用独立章节方案：在“新论文分类与 PDF 归档”之后、“简单总结任务”之前新增“Markdown 阅读报告归档”章节。删除与新规则冲突的“除非用户另有要求，只归档原始 PDF，不自动把论文阅读报告保存为文件。”

## Markdown 归档规则

1. 默认不保存 Markdown 阅读报告。仅当用户主动要求输出或保存 Markdown 文档时，才按照本节规则生成并归档。
2. Markdown 报告的总结内容与输出到飞书文档的版本一致；若此前对话中存在针对论文的问答，将问答内容融合进对应的结构化章节，不单独堆放“问答”章节。
3. Markdown 报告遵循 `AGENTS.md` 的 Markdown 公式规则：所有数学表达使用独立块级 `$$`，开始标记、单行 KaTeX 内容和结束标记分别独占一行；不使用飞书专用的 `<equation>...</equation>` 标签。
4. Markdown 文件与归档 PDF 位于同一目录，文件主名与 PDF 完全一致，仅将 `.pdf` 后缀替换为 `.md`。
5. 若同名 Markdown 文件已存在，先核对内容；内容相同则复用，内容不同则报告冲突，不得静默覆盖。
6. 保存后确认 Markdown 文件非空。保存失败时明确报告原因，不得声称已经归档。
7. 在最终阅读结果中列出 Markdown 报告的本地保存路径。

## 示例

若 PDF 路径为：

`扩散模型/Example Paper_CVPR2026.pdf`

则 Markdown 路径为：

`扩散模型/Example Paper_CVPR2026.md`

## 边界

- 不改变 PDF 的下载、分类、命名与冲突处理规则。
- 不改变默认论文总结结构和简单总结任务的篇幅要求。
- 不要求将飞书专用公式标签写入 Markdown 文件。
- 本次只修改规则文件，不创建实际论文报告。

## 验证标准

- `AGENTS.md` 中存在独立的“Markdown 阅读报告归档”章节。
- 章节明确禁止默认自动保存，并完整覆盖按用户要求保存、内容一致、公式格式、同路径同主名、冲突处理、非空验证和最终路径报告。
- 原有“只归档原始 PDF，不自动把论文阅读报告保存为文件”的独立旧句保持删除，避免与新章节重复表达。
- 不存在“每次生成论文总结时自动保存 Markdown 阅读报告”等相反表述。
- 其他论文阅读与 PDF 归档规则保持不变。
