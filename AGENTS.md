# 论文阅读项目说明

## 项目用途

本项目用于阅读、解析和总结学术论文，重点关注以下方向：

- 计算机视觉与图像恢复
- RAW 图像去噪、低光成像、ISP 与成像噪声建模
- 扩散模型、Flow Matching、MeanFlow 及其改进
- VLM、目标检测与视觉问答
- 图像编辑、智能构图与相关 benchmark

用户通常会提供 arXiv、CVF、OpenReview、项目主页或 PDF 链接，并要求生成结构化中文论文阅读文档。

## 默认工作要求

- 除非用户另有说明，使用中文输出论文阅读结果。
- 内容应清晰、完整，并适合直接复制到文档中。
- 基于论文原文准确总结；区分论文明确结论、作者观点和分析推断。
- 优先解释论文解决的问题、整体流程、模型结构、关键公式、图表、实验结果、创新点、局限性，以及对研究或工程实现的启发。
- 研究方法部分是重点关注内容，需要详细介绍和描述，尽量覆盖论文中与方法相关的所有介绍；不仅概括模块名称，还要解释方法动机、输入输出、模块连接关系、训练或推理流程、关键设计取舍及其作用。
- 关键公式按“公式输出规则”统一处理。
- 若论文提供代码仓库或项目主页，应在基本信息中列出。
- 若用户提出特定问题，优先回答该问题，并按需调整固定结构和篇幅。
- 若用户要求输出到飞书文档，且此前对话中有针对论文的提问与解答，应将这些问答内容融合到对应的结构化章节中，例如研究背景、研究方法、图表分析、主要发现或局限性；不要简单新增一个独立“问答”章节堆放。
- 若用户要求“简单总结”或类似轻量任务，只需简短总结该论文的创新点和主要结论，不必展开完整论文阅读结构。

## 会话命名规则

每次在新会话中阅读论文时，在确认论文官方标题后，自动尝试将当前 Codex 会话标题重命名为论文标题。

- 优先使用论文官方英文标题。
- 若标题过长，可保留核心标题并去掉副标题或冗余说明。
- 若同时阅读多篇论文，则使用主要论文标题，或使用能概括任务的简短中文标题。
- 若当前环境不支持会话重命名或重命名失败，应在最终结果中说明，不影响论文阅读和 PDF 归档。

## 新论文分类与 PDF 归档

每次阅读一篇新论文时，除非用户明确要求不下载，否则自动完成以下操作：

1. 若用户明确指定论文分类，按用户指定的分类归档。指定分类存在明显错别字、歧义，或与现有目录名称近似但不完全一致时，先指出可能对应的分类并等待用户确认；不得自行猜测后创建近义目录。
2. 若用户未指定分类，先阅读论文摘要，并给出若干简洁、明确的推荐分类及推荐理由（例如 `RAW图像去噪`、`扩散模型` 或 `智能构图`）；等待用户确认其中一类或提供新分类。在收到确认前，不下载 PDF、不创建论文归档目录，也不开始后续完整论文总结工作。
3. 分类确认后，优先复用项目中已有的同类目录，避免为近义类型重复创建目录；没有合适目录时，在项目根目录创建以确认后的论文类型命名的目录。
4. 在论文类型目录下，为每篇论文建立独立文件夹。文件夹名称使用论文官方英文标题和发表会议信息，命名规则与 PDF 主文件名一致但不含扩展名；例如 `扩散模型/Simple Baselines for Image Restoration_ECCV2022/`。
5. 获取论文的原始 PDF，并下载到该论文的独立文件夹。下载后确认文件非空且确实为 PDF；下载失败时明确报告原因，不得声称已经归档。
6. 使用论文官方英文标题和发表会议信息重命名 PDF：
   - 已录用或发表：`论文英文标题_会议名年份.pdf`，例如 `Simple Baselines for Image Restoration_ECCV2022.pdf`。
   - 只有 arXiv 信息：`论文英文标题_arXiv.pdf`。
7. 文件夹名和文件名中的 Windows 非法字符 `< > : " / \\ | ? *` 统一替换为连字符，并移除末尾的空格和句点，同时尽量保持标题可读性。
8. 若目标文件或同名论文文件夹已存在，先核对论文内容：内容相同则复用现有目录；内容不同则报告冲突，不得静默覆盖。
9. 在最终阅读结果中明确列出“论文类型”、论文文件夹路径和 PDF 的本地保存路径。

## Markdown 阅读报告归档

默认不保存 Markdown 阅读报告。仅当用户主动要求输出或保存 Markdown 文档时，才按照本节规则生成并归档。

1. Markdown 报告的总结内容与输出到飞书文档的版本一致。若此前对话中有针对论文的提问与解答，应遵循“默认工作要求”，将这些内容融合到研究背景、研究方法、图表分析、主要发现或局限性等对应章节中，不单独新增“问答”章节。
2. Markdown 报告中的公式遵循“公式输出规则”的 Markdown 格式：所有数学表达使用独立块级 `$$`，开始标记、单行 KaTeX 内容和结束标记分别独占一行；不要使用飞书文档专用的 `<equation>...</equation>` 标签。
3. Markdown 文件与归档 PDF 保存在同一篇论文的独立文件夹中，文件主名与 PDF 完全一致，仅将 `.pdf` 后缀替换为 `.md`。例如 `扩散模型/Example Paper_CVPR2026/Example Paper_CVPR2026.pdf` 对应 `扩散模型/Example Paper_CVPR2026/Example Paper_CVPR2026.md`。
4. 若同名 Markdown 文件已存在，先核对文件内容：内容相同则复用现有文件；内容不同则报告冲突，不得静默覆盖。
5. 保存后确认 Markdown 文件非空。保存失败时明确报告原因，不得声称已经归档。
6. 除完整 Markdown 报告外，同时在同一目录保存简要总结文件。其文件名使用完整报告主文件名加 `_sample.md` 后缀，例如 `Example Paper_CVPR2026_sample.md`。该文件仅包含两项内容：核心创新点是什么；主要实验结论或研究结论是什么。不得加入摘要、背景、方法、公式、图表或其他章节。
7. 若同名 `_sample.md` 文件已存在，先核对文件内容：内容相同则复用现有文件；内容不同则报告冲突，不得静默覆盖。保存后确认该文件非空。
8. 在该论文的独立文件夹内创建 `images/` 子文件夹，用于保存从 PDF 截取并在 Markdown 中引用的图像。图像文件使用可读且稳定的名称，例如 `fig_01_method.png`、`table_02_results.png`。
9. 论文中的全部图片应在报告中逐一截取、嵌入并解读，包含所有编号图、子图及其图注信息，不得遗漏。与研究方法直接相关的图片应嵌入第四节对应的方法小节；第五节“图表分析”仅分析未在第四节使用的其余图片和表格，避免重复。第五节应覆盖主要结果表、消融表及其他支撑结论的表格。Markdown 使用相对路径引用 `images/` 中的文件，并在图片前后给出图表编号、简短说明和中文解读；无法清晰截取时，应明确说明原因并保留文字分析。
10. 截图完成后确认每个被引用的图片文件非空且可打开；在最终阅读结果中明确列出完整 Markdown 报告、简要总结文件和图片目录的本地保存路径。

## 简单总结任务

当用户只要求简单总结论文时，输出应简洁，重点包括：

- 核心创新点是什么。
- 主要实验结论或研究结论是什么。

## 公式输出规则

- 当前 Codex Markdown 渲染器不支持在中文段落内使用 `$$...$$`。正文中禁止写“其中 $$z_i$$ 是观测像素”这类行内公式；它会原样显示分隔符。
- 正文尽量用自然语言替代短变量，例如写“目标观测像素”“邻域输入”“网络预测”。如果变量不可省略，先单独给出一行公式，再在下一段用中文解释变量含义；正文中也不要裸写 `f_theta`、`D_bar`、`f_inv(D_bar(f_theta(z)))` 等伪公式。
- 所有数学表达统一使用块级 `$$`。块级公式前后各留一个空行；开始标记 `$$` 和结束标记 `$$` 必须各自独占一行，公式内容也必须只有一行，且该行只能包含 KaTeX。不要在公式内容中插入空行或换行。
- 当前渲染器对多行 KaTeX 块兼容性不稳定。复杂推导必须拆成多个短的、单行块级公式；不要使用 `aligned`、`split`、`array`，也不要将长式拆成多行 `\\`、`\\left`、`\\right` 或续行。
- 禁止混用 `\\[...\\]`、`\\(...\\)`、`$...$` 与 `$$`。正确格式是：先写“将观测模型代入损失函数：”，空一行，单独一行 `$$`，下一行写完整单行公式，再单独一行 `$$`，空一行后继续解释。不要写成“将 $$x_i=s_i+n_i$$ 代入：\\[ ... \\]”。
- 输出到飞书文档时，公式使用独立的 `<equation>...</equation>` 标签，内容保持 KaTeX 兼容；`<equation>` 标签前后各留一个空行，不要嵌入正文段落，也不要用 fenced code block 承载公式。
- 复杂推导优先拆成多个短公式。每个公式前先用中文解释变量含义和公式作用，公式后再说明该公式在方法、训练目标或推理流程中的作用。

## 默认论文总结结构

### 一、论文基本信息

包括：

- 论文标题
- 论文类型
- 会议、期刊或 arXiv 信息
- 作者
- 作者单位
- 发表时间
- 论文链接
- GitHub 或项目主页地址（如有）

### 二、摘要总结

- 约 400–600 字。
- 使用中文重新组织和总结，不直接翻译英文摘要。
- 覆盖研究背景、问题动机、核心方法、主要实验结果和论文意义。

### 三、研究背景

#### 3.1 已有研究进展

说明该方向已有方法、主要技术路线及常见问题。

#### 3.2 具体科学问题

说明论文要解决的具体问题及其重要性。

### 四、研究方法

#### 4.1 数据来源和范围

说明训练集、测试集、数据构建方式、数据规模和适用范围。

#### 4.2 研究方法和模型

详细说明论文提出的方法、网络结构、模块设计、损失函数和训练策略。

这是论文阅读中重点展开的部分。需要尽量覆盖论文中对方法的所有介绍，包括总体思想、核心假设、模型结构、模块设计、算法流程、训练目标、损失函数、优化策略、推理过程、实现细节和关键超参数。

需要展开描述每个核心模块的设计动机、输入输出、内部计算逻辑、与其他模块的连接方式、训练目标、推理过程，以及这些设计如何对应论文要解决的问题。涉及关键公式时，先解释变量含义，再按“公式输出规则”给出数学表达，并说明公式在方法中的作用。若论文方法部分包含多个子章节，应按论文逻辑逐一覆盖，不要只抽象概括。

为便于理解，可在对应的方法小节中嵌入与该模块、流程或训练策略直接相关的论文图片，并在图片前后说明图号、图中组成、数据流和其与文字方法描述的对应关系。这些已在第四节解读的图片不应在第五节重复分析。

#### 4.3 关键分析步骤

按 pipeline 拆解核心流程，解释各阶段的输入、处理和输出。

若论文方法包含多阶段训练、采样、优化、条件控制、数据合成或评估流程，应按步骤详细说明每一步的目的、执行方式、依赖关系和最终产物。

### 五、图表分析

- 仅分析未在第四节“研究方法”中嵌入和解读的剩余图片与表格；不得重复分析已用于方法说明的图。
- 解读每张剩余图片和关键表格展示的内容。
- 说明图中模块、箭头、变量和对比设置的含义。
- 总结图表体现的关键发现与规律。

### 六、主要发现

- 核心科学发现
- 关键实验结果
- 与 baseline 或 SOTA 的对比
- 实验结果是否符合论文预期

### 七、核心贡献

- 方法创新
- 理论贡献
- 数据集贡献
- 工程实现价值
- 对领域发展的推动

### 八、研究局限

- 方法假设的局限
- 数据集局限
- 实验验证不足
- 泛化性问题
- 工程落地风险
- 可行的改进方向

### 九、论文总结

- 总结论文目标、方法、主要发现和贡献。
- 突出论文的核心价值。
- 说明论文对当前研究方向或工程实践的启发。
